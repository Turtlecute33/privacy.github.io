Costruisci il tuo nodo Bitcoin e Lightning network su un raspberry pi con un bel display.

Raspiblitz è un nodo lightning fai da te (LND o c-lightning) che funziona insieme ad un nodo bitcoin su un raspberry pi4 (1TB SSD), con un bel display per un setup semplice e per monitorare le informazioni sul nodo.

L'obiettivo di raspiblitz è di imparare come far girare il proprio nodo in modo decentralizzato a casa - perché: Non il tuo Nodo, Non le tue Regole. Scopri e sviluppa un ecosistema crescente del Lightning Network diventando una parte di esso. Costruiscilo per il tuo negozio o per un tuo progetto di fine settimana.

Tempo stimato per creare un RaspiBlitz

RaspiBlitz è ottimizzato per essere creato ad una officina ad un hackday o conferenza. Quandom arriva completamente assemblato e sincronizzato con la blockchain, è possibile averlo pronto in circa 2 o 3 ore.
Se inizi a casa ordinando tutte le componenti su Amazon (guarda la lista della spesa sotto) allora è un procendo da fare in un fine settimana con molto da scaricare e molto tempo di sincronizzazione dove puoi fare altre cose mentre controlli il progresso di tanto in tanto.
Se già possiedi un nodo con Umbrel, Citadel o myNode hai già tutto l'hardware necessario e puoi effettuare una migrazione a RaspiBlitz in circa un'ora.

Supporto

Se incorri in un problema o hai ancora una domanda, segui i seguenti passi sotto per avere supporto. Inoltre controlla la documentazione del setup per i dettagli. Documentazione

  1. Guarda le FAQ se non puoi trovare una risposta al tuo quesito/problema
  2. Se hai un problema con l'hardware, per favore controlla che le tue componenti siano esattamente le parti raccomandate nella lista della spesa sopra. Diversi schermi o anche custodie per SSD possono causare problemi.
  3. Per favore cerca di capire se il tuo problema è con RaspiBlitz o ad esempio con LND/c-lightning. Per esempio se non puoi pagare o hai un errore quando stai aprendo un canale questo è un problema di LND/c-lightning e le risposte migliori le trovi nella comunità di sviluppo LND o nella documentazione di c-lightning.
  4. Controlla la pagina dei problemi di RaspiBlitz: https://github.com/rootzoll/raspiblitz/issues Fai una ricerca qui. Controlla anche i problemi chiusi rimuovendo 'is:open' dai filtri o dal box di ricerca.
  5. Se non hai ancora trovato una risposta, apri un problema nella pagina RaspiBlitz di GitHub. Potresti doverti prima registrare a GitHub per farlo. Se hai un bug con RaspiBlitz, per favore aggiungi (copia+incolla) un report di debug al tuo problema (guarda le FAQ per capire come generarlo), e/o aggiungi alcuni screenshot/foto così che la community intuisca meglio il tuo problema.

Configurazione di base

Ogni volta che inizi la configurazione con una nuova scheda SD ti verranno proposte diverse opzioni. Per esempio questo è anche il punto dove puoi importare un file di migrazione da un tuo vecchio RaspiBlitz - leggi circa la migrazione più in basso. Ma siccome stai creando un nuovo RaspiBlitz sceglierai l'opzione FRESHSETUP



Quindi ti verrà chiesto cosa bisogna fare con l'HDD/SDD connesso. Se c'è già una blockchain nel tuo HDD/SDD ti verrà chiesto se vuoi usare questi dati già sincronizzati o se va bene eliminarli. Se non ci sono dati della blockchain questa domanda sarà saltata.



In fine dovrai dovrai confermare che tutti gli altri dati nell'HDD/SDD saranno eliminati, ad eccezione della blockchain se l'hai selezionato precedentemente. Questa operazione potrebbe richiedere qualche secondo.



La prima cosa da fare è dare un nome al tuo RaspiBlitz:



Il nome che scegli per il nodo sarà anche utilizzato come alias (nome) pubblico del tuo nodo lightning quindi scegli attentamente. Quindi puoi scegliere quale implementazione di lightning vuoi utilizzare sopra al tuo nodo bitcoin. RaspiBlitz ha iniziato con LND di Lightning Labs che è utilizzato dalla maggior parte dei nodi lightning su raspberry pi e funziona con molte applicazioni aggiuntive. Ma ora puoi anche scegliere CL per c-lightning della Blockstream che è una buona scelta per operatori di nodi più esperti & sviluppatori lightning che vogliono utilizzare un plugin molto personalizzabile che c-lightning offre. È anche possibile usare entrambe le implementazioni di lightning in parallelo al tuo RaspiBlitz successivamente - scegliene uno giusto per iniziare ora. Scegli NONE se sei solo interessato ad utilizzare un nodo bitcoin senza lightning.



Successivamente noi utilizziamo la configurazione utilizzando LND - che è molto simile ai passi da fare con c-lightning
Se hai scelto di utilizzare una delle implementazioni di lightning ti verrà richiesto se vuoi iniziare con un nuovo wallet/nodo lightning o se hai un vecchio wallet lightning/nodo che vuoi ricreare.



Normalmente viene selezionato NEW qui, ma se vuoi recuperare un vecchio wallet hai le seguenti opzioni se selezioni OLD:




LNDRESCUE LND tar.gz-File di backup (MIGLIORE)
Scegli questa opzione se hai già un completo backup dei dati di LND o c-lightning e hai un file tar.gz che inizia con le parole 'lnd-rescue' o 'cl-rescue' disponibile. Ricovererai tutti i tuoi fondi onchain e i canali aperti che avevi.
Ma devi assicurarti che il backup di recupero che hai sia veramente l'ultimo disponibile - altrimenti potresti perdere i fondi che hai sul canale.

Se hai un file tar.gz che inizia con 'raspiblitz', questo è un file di migrazione. Questo include anche il tuo vecchio wallet LND/c-lightning, ma devi importare questo file all'inizio della configurazione con 'FROMBACKUP - Upload Migration Backup' invece di scegliere FRESHSETUP

SEED + SCB Worlds Seed & file channel.backup (OK)
La migliore opzione è se disponi del file channel.backup e del tuo seed. Questo ti permette di recuperare tutti i tuoi fondi onchain (i.e. "importo bitcoin") nel tuo wallet lightning e ti da una buona possibilità di recuperare i tuoi fondi off-chain (i.e. "importo lightning") che avevi nei tuoi canali aperti, finché il tuo nodo remoto supporta la funzione option_data_loss_protect che è molto comune dal 2020. Tutti i canali che avevi aperto prima saranno chiusi durante questa procedura. Guarda Bitcoin Optech - Static Channel Backups per migliori dettagli circa le informazioni di background di questo processo.

SOLO SEED Solo seed dalla lista di parole (ripiego)
Se hai solo la lista di parole del seed (Dalla versione RaspiBlitz 1.1 e precedenti) puoi almeno recuperare i tuoi fondi onchain. Il recupero dei fondi nei canali è molto improbabile in questo scenario.
Ma normalmente stai creando un nuovo nodo - quindi scegli semplicemente l'opzione NEW in questo menu



In fine dovrai impostare tre password chiamate A, B & C. Scegli ogni password in modo unico, una sola stringa, senza spazi né caratteri speciali, e che sia almeno lunga 8 caratteri. Puoi usare questo Foglio di recupero RaspiBlitz (PDF) per scrivere queste password per un salvataggio sicuro e anche usarle dopo per il tuo seed.
Inanzitutto, viene richiesta la password A: questa è la password che userai per accedere via SSH ed è la password impostata per qualunque utente di default: admin, root, bitcoin & pi.
Bitcoin e i servizi lightning saranno successivamente girati in background (come demoni) e usano l'utente separato "bitcoin" per motivi di sicurezza. Questo utente non ha diritti di amministrazione e non può cambiare le impostazioni del sistema.
Dopo inserisci la password B: questa è usata internamente per l'interfaccia RPC di bitcoin. È anche utilizzata come modalità d'accesso ad applicazioni come RTL-WebGUI o il blockexplorer.
Infine inserisci la password C: questa viene utilizzata per per criptare/bloccare il wallet lightning nell'HDD/SDD ed è utilizzata per LND. Ogni volta che si avvia/riavvia il nodo lightning LND ha bisogno di caricare il wallet nella memoria per lavorarci e ti chiederà la password C per "sbloccare" il wallet.
Nelle versioni precedenti di RaspiBlitz era anche presente un'altra password D, che non viene più utilizzata.

Se non hai un'intera copia della blockchain già sincronizzata/valitata nel tuo HDD/SDD allora ti verrà richiesto ora come vuoi ottenere una copia della blockchain. Ci sono due opzioni di base:



1. SINCRONIZZAZIONE - Valida da solo tutti i Blocchi
Per il nuovo raspberry pi 4 (con SSD e almeno 2gb di RAM) questo è il miglior modo per farlo. Richiederà circa 3-6 giorni per sincronizzare e validare i blocchi direttamente con la rete di bitcoin. Con questa opzione, lo hai fatto nel modo ufficiale "don't trust, verify (non fidarti, verifica)".
Per il vecchio raspberry pi 3 questo non è consigliato. Un raspberry pi 3 è una potenza della CPU molto bassa e l'operazione di sincronizzare+validare la blockchain in modo diretto con la rete peer2peer può richiedere diverse settimane: questo è il motivo per cui per un raspberry pi 3 dovresti scegliere l'opzione di copia.

2. COPIA - Copia dal tuo laptop o da un altro RaspiBlitz utilizzando la rete locale
Se hai un amico che sta già girando un RaspiBlitz sincronizzato o se hai un laptop con abbastanza spazio libero nell'hard disk da poter sincronizzare & validare la blockchain molto più velocemente puoi anche scegliere l'opzione COPY (opzione di copia). Puoi quindi eliminare la blockchain esistente che RaspiBlitz aveva già iniziato a sincronizzare per te



Per copiare da un altro RaspiBlitz scegliere BLITZ e segui le istruzioni. Sappi che l'altro Blitz sarà offline dal lightning network durante la copia che potrebbe richiedere diverse ore.
Per copiare dal tuo laptop/computer (opzione per WINDOWS, MACOS, & GNU/LINUX) avrai prima bisogno di scaricare e validare la blockchain nel tuo computer/laptop. Per farlo, installa l'ultima versione di bitcoin-core (avrai bisogno di circa 400GB). Quindi sotto l'opzione COPY scegli il sistema operativo. La copia verrà fatta sotto la rete locale da SCP (SSH file transfer): segui le istruzioni date nei dialoghi. È consigliabile mantenere un backup un backup di Bitcoin Core & la cartella "data" della blockchain nel tuo laptop/computer nel caso avrai bisogno di riconfigurare il RaspiBlitz.
Più dettagli: Ho l'intera blockchain in un altro computer. Come la copio nel raspiblitz?
Se non hai già la blockchain di bitcoin in un altro laptop o in un RaspiBlitz scegli semplicemente SELFSYNC.
Ed evvia :D Il tuo RaspiBlitz è pronto all'uso! Benvenuto nuovo operatore di un nodo.



Se premi OK, il RaspiBlitz farà un riavvio finale.



Aspetta giusto un altro po' e poi dai il comando SSH per accedere di nuovo. Accedere via SSH richiede la password A che hai impostato precedentemente.

Il tuo RaspiBlitz potrebbe richiedere un po' di tempo per sincronizzare e validare la blockchain: possono essere diversi giorni. All'inizio potresti vedere veloci progressi ma questi diventeranno più lenti in quanto i blocchi inizieranno ad essere più pieni. La CPU del tuo RaspberryPi diventerà molto calda durante la sincronizzazione iniziale. Tuttavia questo va bene in quanto il RaspberryPi ha delle protezioni contro il surriscaldamento e questo assicura che la CPU non sarà surriscaldata in modo critico.

Sotto "LND/c-lightning Wallet options" troverai alcuni tool di base per gestire il tuo nodo lightning. Questi tool sono molto semplificati nel menu SSH di RaspiBlitz per lo scopo di imparare. Per una gestione più avanzata del tuo nodo lightning guarda le applicazioni addizionali sotto SERVICES.

FINANZIAMENTO: Finanzia il tuo wallet onchain
Prima di aprire canali lightning con altri nodi hai bisogno di mettere alcune monete nel tuo wallet onchain (controllato dal tuo wallet lightning). Usa questa opzione per generare un indirizzo dove mandare fondi.

Ricordati: RaspiBlitz è un software sperimentale. Con l'opzione di finanziare il tuo nodo lightning accetti il rischio di perdere fondi. Quindi gioca solo con piccoli importi: qualcosa come 20 euro o dollari potrebbero essere abbastanza per fare la tua prima esperienza. Inoltre, è una buona pratica per la privacy di fare coinjoin dei tuoi coin (monete) prima di mandarli nel wallet lightning network.

Puoi finanziare il tuo wallet diverse volte: iniziando con piccoli importi per fare prove. Il tuo nodo lightning genererà sempre un indirizzo differente, ma tutti i fondi che riceverai saranno sempre ricevuti dallo stesso wallet onchain LND.

CONNESSIONE: connettiti ad un Peer
Prima di poter aprire un canale con un altro nodo nella rete, hai bisogno di connetterti a questo nodo come un peer al tuo nodo.
Aprire un canale con un peer è una cosa opzionale. Avere altri nodi come peer aiuta il tuo nodo a ricevere informazioni circa il lightning network attraverso il protocollo gossip. Aiuterà il tuo nodo a trovare migliori route attraverso il network.
CANALE: aprire un canale con un Peer
Per aprire un canale di pagamento con un altro nodo, puoi utilizzare questa opzione.
Puoi trovare interessanti nodi per aprire un canale attraverso siti come https://1ml.com/ o entrare nel gruppo telegram "RaspiBlitz NodeManager" per incontrare persone con cui aprire canali con: https://t.me/raspiblitz
Tieni a mente che questa opzione aprirà un canale pubblico che può essere visto da chiunque altro nella rete e scelto dagli altri come parte della via (route) di pagamento. Questo è buono se vuoi fare da routing per i pagamenti. Se non vuoi fare da routing per i pagamenti agli altri, puoi scegliere l'opzione così chiamata "canale privato (non annunciato)" che altri non possono usare come strada per il loro pagamento. Per fare questo devi utilizzare la linea di comando e aprire un canale con l'opzione "-private".

Gira sotto Tor
Puoi girare sia il tuo nodo Bitcoin & Lightning, come anche le applicazioni addizionali, sotto un "hidden service" (servizio nascosto) Tor. Questo rimpiazza il tuo IP in clearnet con un indirizzo ".onion".



Girare il tuo nodo come hidden service ha molti benifici:
  • Non pubblichi il tuo indirizzo IP e quindi è quindi molto più difficile risalire al tuo vero nome e alla tua posizione
  • Fai tunnel attraverso il NAT e rendi il tuo nodo Bitcoin e Lightning raggiungibili (permettendo connessioni in entrata) da tutti gli altri nodi della rete Tor.
  • Utilizzando un indirizzo Tor è possibile muovere il tuo nodo ad un indirizzo IPv4 diverso e mantenere esistenti (precedentemente aperti e finanziati) le funzionalità dei canali.

Ma questo ha anche alcuni svantaggi:
  • Alcuni wallet mobile non supportano ancora la connessione ad un RaspiBlitz via Tor
  • I nodi lightning che non girano sotto Tor non possono raggiungerti (come sotto NAT)


Usare Tor nasconde il tuo indirizzo IP ma aumenterà il tempo che ci mette il tuo nodo ad instradare un pagamento. Impostare un nodo su clearnet ti darà un tempo di risposta più basso quando instradi un pagamento, ma il tuo indirizzo IP sarà disponibile al resto della rete come il nodo https://1ml.com/node/03c2abfa93eacec04721c019644584424aab2ba4dff3ac9bdab4e9c97007491dda. Se hai bisogno di privacy o non vuoi fare ddos la tua rete domestica allora Tor potrebbe fare al caso tuo. Tuttavia se la privacy non è una cosa di cui hai bisogno e/o vuoi, o stai provando a creare un servizio per cui il tuo nodo fa da backend, allora la clearnet potrebbe essere la scelta più vantaggiosa.

Per provare la rete Tor, accendi il servizio: lo potrai disattivare se non fa per te.

Backup statico dei canali con SCP/SSH in un altro server
Un opzione per utenti più avanzati -- che puoi selezionare direttamente nel file "raspiblitz.conf" -- è il backup automatico del backup statico dei canali in un altro server utilizzando SCP/SSH. Per questo devi scegliere il valore:
scpBackupTarget='[USER]@[SERVER]:[DIRPATH-WITHOUT-ENDING-/]'
e puoi opzionalmente impostare un opzione personalizzata per il comando SCP (per esempio per impostare una porta non di default) con:
scpBackupOptions='[YOUR-CUSTOM-OPTIONS]'
Nel server di destinazione aggiungi la chiave pubblica dell'utente root del tuo RaspiBlitz nel file dell'utente "authorized_keys" - per fare questo guarda: https://www.linode.com/docs/security/authentication/use-public-key-authentication-with-ssh/

ISCRIVITI: Servizi ad iscrizione
Il RaspiBlitz offre ora anche le iscrizioni. Queste potrebbero essere gratuite o da pagare per servizi terzi.



Sotto "LIST my Subscriptions" troverai sempre la lista di tutte le tue sottoscrizioni attive, per avere più dettagli o per fare "CANCEL" (cancellarle).
Al momento, sono disponibili i seguenti servizi ad iscrizione:

IP2TOR (a pagamento)
IP2TOR è un servizio di tunneling dove puoi girare il tuo RaspiBlitz sotto Tor in modo anonimo ma affittare una porta per un IP in clearnet in modo che tu possa rendere i tuoi servizi del RaspiBlitz facilmente disponibili per tutti attraverso Internet. Non hai bisogno di interessarti del tuo router o delle impostazioni del firewall. Puoi pagare per questo servizio direttamente attraverso Lightning dal tuo RaspiBlitz come un abbonamento.
In primo luogo devi selezionare che servizio del tuo RaspiBlitz desideri farlo passare per un tunnel ad un bridge IP2TOR.
Otterrai una lista di tutti i bridge IP2TOR disponibili. Seleziona "OK" su un'offerta per un bridge IP2TOR e ne vedrai i dettagli, come quanti satoshi ti costerà l'iscrizione. Il tuo nodo dovrebbe essere finanziato e avere già dei canali aperti a questo punto.
Se scegli "AGREE" nei dettagli di un bridge IP2TOR il RaspiBlitz proverà per te una configurazione del bridge IP2TOR. Se tuttoquanto funziona troverai la tua nuova iscrizione sotto "MAINMENU" > "SUBSCRIBE" > "LIST my Subscriptions" dove la potrai ancora cancellare.
Per provare il tunnel IP2TOR scegli nel "MAINMENU" il punto del menu extra del servizio per cui hai scelto il bridge, e ti darà un URL aggiornato o proverà a chiamare le API nell'IP e la porta che viene mostrata sotto i dettagli dell'iscrizione nella sezione "LIST my Subscriptions".

HTTPS con LetsEncrypt (gratis)
Se vuoi un servizio web, come un BTCPayServer o LNbits, per essere disponibili su Internet (tipo come IP2TOR), le persone si aspettano che tu offra un indirizzo HTTPS così le comunicazioni tra il client e il tuo RaspiBlitz siano criptate. Potresti usare un certificato HTTPS firmato da te che il RaspiBlitz ti offre, ma questo darà agli utenti degli avvisi di sicurezza nel loro browser, il che non è molto intuitivo e semplice.
Qui è dove puoi utilizzare un'iscrizione a Let's Encrypt per avere un certificato HTTPS gratuito che è accettato senza avvertimenti per quasi tutti i browser comuni.
Siccome hai anche bisogno di un dominio per questo avrai bisogno di aprire un account gratuito, i seguenti sono supportati, sarebbe bello aggiungerne altri con l'aiuto della community: DuckDNS.org DYNU.com (AL MOMENTO NON DISPONIBILE).
Quando crei una nuova sottoscrizione a LetsEncrypt, ti verrà richiesto il tuo sottodominio su DuckDNS e il codice di autenticazione del tuo profilo DuckDNS. Il RaspiBlitz proverà a configurare tutto al posto tuo. Se tutto ha funzionato, troverai la tua iscrizione sotto "MAINMENU" > "SUBSCRIBE", "List My Subscriptions" , dove puoi cancellare l'iscrizione in qualsiasi momento se vuoi.
Per provare il tunnel IP2TOR, vai nel "MAINMENU" e il punto del menu aggiuntivo del servizio per la quale vuoi usare il bridge. Ti dovrebbe dare un URL aggiornato per chiamare il tuo servizio.
HTTPS può essere un argomento spinoso, puoi provare un checker SSL per trovare velocemente problemi con l'installazione del tuo certificato SSL: https://www.sslshopper.com/ssl-checker.html

Creare un RaspiBlitz da Umbrel, Citadel o MyNode


Un altro tipo di migrazione è quando hai già un full node con Umbrel, Citadel o MyNode e vorresti passare a RaspiBlitz senza chiudere i canali. Hai già tutto l'hardware necessario e vuoi solo cambiare software. Non hai bisogno del display LCD: RaspiBlitz può funzionare anche senza di esso.
NOTA: La migrazione è ancora sperimentale. Dovrai ancora usare il terminale per configurare e gestire il tuo RaspiBlitz. Se questo è troppo tecnico per te per favore aspetta fino alla versione di RaspiBlitz 1.8, che introduce una interfaccia grafica web per il tuo browser che renderà la migrazione a RaspiBlitz più familiare alla tua esperienza con il vecchio nodo.

Prima di iniziare la migrazione:
  • Potresti avere fondi onchain nel tuo vecchio nodo: assicurati di avere un backup del seed
  • se hai dei canali lightning aperti con il tuo vecchio nodo: assicurati di aver scaricato l'ultima versione del backup statico dei canali sul tuo laptop.

Inoltre sappi che al momento RaspiBlitz può solo trasferire la tua blockchain e i tuoi dati di LND (inclusi i canali) sul tuo RaspiBlitz. Altri dati/accoppiamenti o applicazioni addizionali non possono essere trasferiti e potrebbero essere persi.
Istruzioni per la migrazione:

  • Spegni il tuo vecchio nodo
  • rimuovi la scheda sd
  • Scarica l'ultima versione dell'immagine della scheda sd & caricala nella tua scheda sd
  • Inserisci la scheda sd, avvia & fai l'accesso tramite SSH.


Ora RaspiBlitz dovrebbe mostrarti che i vecchi dati del tuo nodo sono stati trovati e ti offrirà di fare una migrazione per te.

  • Se scegli di continuare preparerà i dati e farà un reboot
  • La modalità di ripristino/aggiornamento di RaspiBlitz sta iniziando: questa operazione richiede un po' di tempo e un riavvio
  • Accedi via SSH come prima e reimposta le password ("FINAL RECOVER LOGIN" nell'LCD). Quindi ci sarà un riavvio finale
  • Accedi via SSH con la tua password A & sblocca il wallet LND con la password C. Ora la blockchain deve recuperare il ritardo e dopo il tuo RaspiBlitz dovrebbe essere pronto e mostrarti (sotto "INFO") i tuoi fondi onchain & i fondi nel canale.


Se non hai un display LCD o un monitor HDMI la connessione potrebbe essere un po' difficile per vedere in che stato si trova il tuo RaspiBlitz. Riprova ad accedere ancora via SSH dopo un riavvio. Può richiedere un po' di tempo dopo un riavvio prima che il sistema operativo e il servizio SSH si avvii, quindi fai diverse prove ad intervalli fino al successo.

Backup per fondi onchain & fondi del canale
Da LND v0.6 (e RaspiBlitz v1.2), è disponibile una funzione chiamata "backup statico del canale". Con RaspiBlitz questa è utilizzata con il file "channel.backup".
In questo momento è la strada migliore per fare un backup e proteggere i tuoi fondi del canale su RaspiBlitz, quindi è raccomandato usarla.
Per recuperare i tuoi fondi in questo modo sono necessarie due cose:

  • Le 24 parole del seed
  • l'ultimo file channel.backup


Dovresti aver scritto le parole del seed durante la configurazione del wallet, tienile in una posizione sicura offline. Il file "channel.backup" è mantenuto nell'HDD (sotto /mnt/hdd/lnd/data/chain/bitcoin/mainnet/) e aggiornato da LND ogni volta un nuovo canale è aperto o chiuso. L'ultima versione del file è necessaria per recuperare tutti i tuoi fondi. Nel caso il tuo HDD sia stato danneggiato RaspiBlitz mantiene sempre una seconda copia dell'ultima versione del file "channel.backup" nella scheda SD: "/home/admin/backups/scb/" .
Se vuoi andare uno step oltre nel mondo in cui metti in sicurezza i tuoi fondi contro una perdita totale del tuo RaspiBlitz (viene completamente danneggiato, rubato o perso), allora puoi addizionalmente impostare una postazione al di fuori di esso, o un backup sul cloud del file "channel.backup". Il file in se è criptato dal tuo seed quindi è accettabile salvare una copia del file su terze parti non affidabili per una funzione di backup (se vuoi).
Il video spiega in dettaglio come è possibile impostare più indietro le informazioni del backup del canale statico:

A) Nextcloud
Nextcloud è un progetto open source per hostare i tuoi file: https://en.wikipedia.org/wiki/Nextcloud: in modo basico è un rimpiazzo aperto di DropBox... ma può fare molto di più. Puoi girarlo te stesso o usare un server di Nextcloud già hostato da terzi.
Puoi trovare provider di Nextcloud per registrarti in modo gratuito qui: https://nextcloud.com/signup/

2) Target di backup SCP
Puoi anche fare il backup del backup statico dei canali sul tuo server, ma questo ha bisogno di una configurazione manuale:

Nel "/mnt/hdd/raspiblitz.conf" il parametro "scpBackupTarget='[USER]@[SERVER]:[DIRPATH-WITHOUT-ENDING-/]'" può essere impostato per abilitare questa caratteristica. Su un server remoto, la chiave pubblica dell'utente root di RaspiBlitz ha bisogno di essere aggiunta al file "authorized_keys" in modo che non ci sia bisogno di una password per lo script in background per fare il backup.
Lo script "/home/admin/config.scripts/blitz.ssh.sh" mostra ("root-get") e trasferisce ssh-pubkey ("root-transfer") ad un server remoto.
Per fare una prova, prova ad aprire o chiudere un canale e controlla se trovi una copia del channel.backup nel tuo server remoto. Puoi controllare i log dello script in background per guardare dettagli sugli errori: "sudo journalctl -f -u background"

Crea l'immagine per la scheda SD


Un'immagine pronta per l'utilizzo del RaspiBlitz è fornita da noi per essere scaricata, per far in modo che tutti possano iniziare presto (guarda sotto). Ma se vuoi creare l'immagine da solo - ecco una veloce guida:
  • Scarica l'ultima versione di RaspiOS 64-bit
  • Scrivi l'immagine sulla tua scheda SD
  • aggiungi un file chiamato "ssh" nella directory radice della scheda SD quando viene montata sul tuo laptop per abilitare il login tramite SSH.
  • Inizia la scheda nel Raspi e accedi via SSH con "ssh pi@[IP-OF-YOUR-RASPI]". La password è "raspberry"

Ora sei pronto per iniziare lo script per fare la build sulla SD (controlla il codice per vedere se l'installazione e la configurazione sono OK per te). Copia il seguente comando nel tuo terminale ed esegui:
"wget https://raw.githubusercontent.
com/ rootzoll/raspiblitz/v1.7/
build_sdcard.sh &&
sudo bash build_sdcard.sh"
Come puoi vedere dall'URL, puoi trovare lo script per la build in questa repo di Git sotto "build_sdcard.sh".
Puoi controllare cosa viene installato e configurato nel dettaglio con le opzioni opzionali. Sentiti libero di caricare miglioramenti con una pull request.
L'intero processo di build potrebbe impiegare del tempo. Alla fine i driver LCD saranno installati e un riavvio sarà necessario. Un utente "admin" è creato durante il processo. Ricordati che la password di default ora è "raspiblitz". Puoi fare login con SSH ancora - questa volta usa admin: "ssh admin@[IP-OF-YOUR-RASPI]" . Il dialogo di installazione del RaspiBlitz dovrebbe partire in modo automatico. Se non vuoi continuare con l'installazione in questo momento, e usare questa scheda SD come modello per configurare diversi RaspiBlitz, clicca "Cancel" a gira "release". Una volta che vedrai l'LCD diventare diventare bianco e il LED di attività del raspberry diventare scuro puoi staccare la corrente e rimuovere la scheda SD. Ora hai costruito la tua immagine RaspiBlitz per la scheda SD.

NOTA: se stai pianificando di utilizzare la tua build per la scheda SD come una copia principale e distribuirla: usa una piccola scheda SD da 8gb per questo. In questo modo sarà assicurato che entri in tutte le schede SD da 16gb raccomandato per il RaspiBlitz dopo.
