Guide to De-Google and privacy on android

Last update 13/12/2021

Operating system

There are various android privacy oriented operating systems (GrapheneOS, CalyxOS, etc) in this guide however we will treat the approach using as base the LineageOS 12 (with microG) but is still compatible with any other AOSP operating system.

!WARNING!

I don't recommend this guide if you use GrapheneOS as they don't recommend rooting the device, which we will do (a non-rooted phone is safer because apps with root permissions can have full access to the device, any rooted apps in this guide are completely opensource and verified, so we can consider them safe, it's a compromise to have more freedom to block hidden trackers and privacy holes in the system).


To simplify the steps for installing LineageOS I'll make a very brief summary:

  1. Unlock bootloader
  2. Flash one recovery
  3. Flash Lineage OS microG version

At this point we will find ourselves with a device with a freshly installed operating system.

Editing and setup of operating system

To have a privacy oriented device we will start by modifying the various settings of the operating system, here I list the main settings and tricks even if they can change from device to device:

  • Disable bluetooth and localization whenever you don't use them
  • Enable privacy in the lock screen and a method of device protection (hidden pins and notifications with locked phone)
  • Disable all telemetry data
  • Enable device encryption if disabled
  • Disable backups
  • Refuse, whenever possible, to share telemetry and crash data in any app you install
  • Remember, if you use it, to disable the usb debugging (disabled by default) because if you lose the phone it exposes you to great risks in the security field

We'll manage application permissions later instead.

Magisk,Riru and modules

We proceed with the installation of magisk, some people prefer not to root as a rooted device is less secure, we in this guide will only use open-source applications that we are sure can not harm us with permissions administrator permissions.

Installed magisk (to be flashed in recovery) and magisk manager (apk to manage the root) we also install the following modules:

  • Magisk hide (will help us to hide the root from apps such as banking apps that often have blocks inserted
  • RIRU (ART and DALVIK memory modifier)
  • RIRU LsPosed (implementation of xposed)
  • Systemless host (you enable it from magisk settings and it will allow us to block trackers, advertising and loggers in combination with other apps that we will install later)

After rebooting the device we open LsPosed and install the following modules:

  • Android faker (application that allows us to change automatically or with a single click all the unique ids I recommend every so often (once a week/month) to open the app and click on random all to apply the new IDs. click on random all to apply the new IDs because occasionally does not apply them at startup.

  • XprivacyLUA (an app that will block the hidden permissions that apps take automatically and instead of providing the data instead of the data requested by them, it will provide false and random information to protect our security and privacy.

Managing app permissions

As a first step we go to our phone's settings and go to apps → advanced → permission management and here we remove any unnecessary app permissions (by unnecessary I mean that for example, an app like whatsapp, needs our contacts if we want to find them, so for the proper functioning of the application it's better to leave this permission, but we can safely disable access to messages as it will be used only the first time when we'll use it. as it will only be used the first time when we login to speed up the first access, after which if left active it gives whatsapp the ability to spy on all our SMS).

We pay special attention to geolocation, camera and microphone permissions.

When an app asks us for a permission that is not useful or you are aware that you occasionally use this service grant the permission 'just this once' so that you don't have continuous access to e.g. location.

After modifying all the permissions we are also going to disable internet access to all the applications that do not need the network (for example the various file managers, calculators, diaries, notes, keyboards, etc.), we can do it comfortably going in the information of an application → data and network → disable 'network access'.

In case you want to be further sure that these apps are blocked from the internet you can use services like AFWALL to block them behind a firewall, don't underestimate the internet access of apps, many, despite not needing Don't underestimate the internet access of apps, many of them, despite not needing it to work, use it all the time to share our data.

First of all we go to the 'service' section and through the menu at the top right click on 'search components in all apps' and search for the following words:

  • AdService and block all services that are named exactly like this

Finally, let's go to activity → top right menu → search in all components and search:
  • Google and disable all services googleAPI except those of microG and aurora strore.

Occasionally look to see if installing new apps has new services or activities to block.

In case an app crashes at boot time, privately reactivate some of the blocked services in order to find out what's causing it to crash, and if that doesn't work either, try reinstalling it.

Warden

This app made by the same creator of the Aurora store is great for blocking all trackers quickly and automatically, Warden scans all installed apps (enable from settings the scan of system apps as well) and automatically blocks all trackers found (it compares them offline with the exodus privacy database, you can disable internet access to the app), occasionally like with my android tool it's worth re-scanning apps to block trackers of apps installed in the meantime.

To block all trackers open the app, enable the system app scan in the settings, open the side menu on the left and go to 'labs', then start the service 'nuke it all'; wait a few minutes and all the trackers found in the minute and all trackers found in our apps will be blocked.

Cloud and Vpn

I'm doing a little separate section for these two subtopics: The best cloud solution would be self hosting with nextcloud (in my opinion the best choice ), in case this is not possible the solution in my opinion easier and faster is Mega.nz (20gb free and 5gb bandwith and accepts payment in bitcoin for pro plans) if we want additional security we can use a cloud with support for WebDav as icedrive in combination with Cryptomator (program that encrypts files before uploading to the cloud).

The VPN discourse instead is particular, a vpn theoretically encrypts our traffic and makes it pass through a node putting itself between us and our ISP so giving us privacy to it but forcing us to trust the vpn service itself.
All companies in this area promise privacy and no logging policies but virtually 99% of VPNs more than give us privacy we lose it, the only one that I really feel like recommending is Mullvad (onion link), it is of course mandatory to pay in bitcoin for more privacy.
